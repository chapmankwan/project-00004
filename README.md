# SaaS Infrastructure — TypeScript / Next.js / Vercel / Supabase

## Stack

| Layer | Tool |
|---|---|
| Framework | Next.js 14 (App Router) |
| Styling | Tailwind CSS + shadcn/ui |
| API | tRPC |
| Auth | Clerk |
| Database | Supabase (PostgreSQL) |
| ORM | Drizzle |
| Cache | Upstash Redis (optional) |
| Background jobs | Inngest |
| Email | Resend + React Email |
| Billing | Stripe |
| Hosting | Vercel |
| Monitoring | Sentry + Axiom |

---

## First-time setup

### 1. Local environment

```bash
cp .env.local.example .env.local
# Fill in your dev/test keys
```

### 2. Install dependencies

```bash
pnpm install
```

### 3. Run database migrations

```bash
pnpm drizzle-kit migrate
```

### 4. Start local dev

```bash
# Terminal 1 — Next.js
pnpm dev

# Terminal 2 — Inngest dev server (background jobs)
npx inngest-cli@latest dev

# Terminal 3 — Stripe webhook forwarding
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

## Terraform (infrastructure provisioning)

> Run once to provision Vercel project + Supabase project.
> After that, Vercel deploys automatically on git push.

```bash
cd terraform

# Copy and fill in your secrets
cp terraform.tfvars.example terraform.tfvars

# Init providers
terraform init

# Preview changes
terraform plan

# Apply
terraform apply
```

### Required accounts / tokens

| Service | Where to get token |
|---|---|
| Vercel | vercel.com/account/tokens |
| Supabase | app.supabase.com/account/tokens |
| Clerk | dashboard.clerk.com → API Keys |
| Stripe | dashboard.stripe.com/apikeys |
| Resend | resend.com/api-keys |
| Inngest | app.inngest.com → Manage → Keys |
| Sentry | Your Sentry project → Settings → DSN |

## GitHub Actions (CI/CD)

The `.github/workflows/ci.yml` workflow runs on every push and PR:

1. **Lint + typecheck + tests** — on every push and PR
2. **DB migrations** — on merge to `main` only
3. **Deploy** — Vercel handles this automatically via the GitHub integration

### Required GitHub secrets

Set these under `Settings → Secrets and variables → Actions`:

- `DATABASE_URL` — Supabase connection pooler URL

### Required GitHub variables

- `PRODUCTION_URL` — your production domain

## Project structure

```
src/
  db/
    schema/        # Drizzle table definitions
    migrations/    # Auto-generated by drizzle-kit
  env.ts           # Zod-validated environment variables
  server/
    routers/       # tRPC routers
  app/
    api/
      trpc/        # tRPC HTTP handler
      webhooks/    # Stripe + Clerk webhooks
  inngest/         # Background job functions
```
